## Key Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Next.js 16 + React 19 | Cutting-edge with `use cache` directive for server-side caching |
| NextAuth v5 (beta) | Native App Router support, edge-compatible middleware auth |
| JWT sessions (not DB sessions) | Stateless, no session DB needed, 30-day expiry |
| Mongoose ODM | Schema validation at DB layer, middleware hooks, TypeScript interfaces |
| Tailwind CSS v4 | PostCSS-first, native CSS features, no `tailwind.config.js` needed |
| Server Actions for reads | Co-located data fetching, automatic serialization |
| API Routes for writes | Full request/response control for auth flows, rate limiting |
| In-memory rate limiting | Simple approach for single-instance deployment |
| PostHog proxy via rewrites | Bypasses ad blockers by routing through same domain |
