## Overview

**DevEvent** is a developer event discovery and booking platform built with Next.js 16 (App Router) and React 19. Users can browse tech events, book spots via email, and authenticate via credentials or Google OAuth. The platform uses MongoDB Atlas for persistence, Resend for transactional emails, Cloudinary for image hosting, and PostHog for product analytics.

### Middleware (`middleware.ts`)

Role-based route protection using NextAuth edge middleware:
- **Auth pages** (`/login`, `/signup`, `/forgot-password`) → redirect if already authenticated
- **Protected routes** (`/profile`, `/dashboard`) → redirect to login if unauthenticated
- **Admin routes** (`/admin/*`) → requires `admin` role
- **Organizer routes** (`/organizer/*`) → requires `organizer` or `admin` role
- Excludes: static files, `_next/`, `favicon.ico`, `/ingest` (PostHog), non-auth API routes

## Data Flow

1. **User visits page** → Middleware checks auth/role → allows or redirects
2. **Home page** (`/`) → Server Component calls `getAllEvents()` (server action) → MongoDB query via Mongoose → cached response via `use cache`
3. **Event detail** → `getEventBySlug()` → renders `EventDetails` + `BookEvent` components
4. **Booking** → `BookEvent` form → `createBooking()` server action → validates eventId exists → creates Booking document
5. **Signup** → POST `/api/auth/signup` → Zod validation → bcrypt hash → create User + EmailVerificationToken → send Resend email
6. **Login** → NextAuth Credentials provider → `authorize()` → bcrypt compare → JWT token → session
7. **Google OAuth** → NextAuth Google provider → `signIn` callback → link/create User → JWT with MongoDB `_id`
8. **Email verification** → click link → POST `/api/auth/verify-email/confirm` → hash token → find doc → mark emailVerified
9. **Password reset** → POST `/api/auth/forgot-password` → generate token → send email → user clicks → POST `/api/auth/reset-password`

## Conventions

**Naming:** PascalCase for components, camelCase for utilities and actions, kebab-case for model files  
**Structure:** Feature-based routing via App Router, centralized database layer, Zod validation schemas co-located in `lib/validations/`  
**Auth pattern:** Edge-safe config split (`auth.config.ts` + `auth.ts`), JWT sessions with custom claims (roles, emailVerified)  
**Data access:** Server Actions for page data, API Routes for auth flows  
**Styling:** Tailwind CSS v4 via PostCSS, Google Fonts (`Schibsted_Grotesk`, `Martian_Mono`), dark theme  
**Error handling:** Consistent `{ success, data?, error? }` return pattern in server actions
