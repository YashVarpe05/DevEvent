# Plan 4.2: Organizer Attendee Management APIs

## Objective
Implement organizer-facing API routes to fetch attendee lists, export data, and mark attendees as checked in.

## Context
- `database/registration.model.ts`
- `app/api/organizer/`

## Tasks

<task type="auto">
  <name>Implement Organizer Attendee List & Export Routes</name>
  <files>
    - app/api/organizer/events/[id]/attendees/route.ts
    - app/api/organizer/events/[id]/attendees/export/route.ts
  </files>
  <action>
    - **GET `/api/organizer/events/[id]/attendees`**: Auth required. Must be event owner or admin. Return paginated list of `Registration` documents for the given `eventId`, sorting by newest first. Allow filtering by `status`.
    - **GET `/api/organizer/events/[id]/attendees/export`**: Generate and return a CSV format string of attendees (Name, Email, Status, Date, Check-in Time, Ticket Code) for the specific event.
  </action>
  <verify>Routes exist and handle authorization explicitly.</verify>
  <done>APIs return correct JSON and CSV formats for authorized organizers.</done>
</task>

<task type="auto">
  <name>Implement Organizer Check-in Route</name>
  <files>
    - app/api/organizer/registrations/[id]/check-in/route.ts
  </files>
  <action>
    - **POST `/api/organizer/registrations/[id]/check-in`**: Mark a registration as checked in.
    - Auth required. The user must be the organizer of the linked event.
    - Set `checkedInAt` to `new Date()` and `checkedInBy` to the current user's ID.
    - Error if already checked in or if status is cancelled.
  </action>
  <verify>Route correctly validates current status before updating.</verify>
  <done>Endpoint successfully updates the `checkedInAt` timestamp.</done>
</task>

## Success Criteria
- [ ] Organizer can fetch a paginated/filtered list of attendees.
- [ ] Organizer can download a CSV of attendees.
- [ ] Organizer can check-in an attendee securely.
