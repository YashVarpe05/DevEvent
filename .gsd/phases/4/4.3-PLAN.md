# Plan 4.3: Attendee UI - Registration Flow

## Objective
Integrate the registration APIs into the frontend so attendees can seamlessly claim free tickets and view their bookings.

## Context
- `app/events/[slug]/page.tsx`
- `components/`

## Tasks

<task type="auto">
  <name>Update Event Detail CTA</name>
  <files>
    - components/BookEvent.tsx
    - app/events/[slug]/page.tsx
  </files>
  <action>
    - Refactor `BookEvent.tsx` to handle the new logic.
    - State 1: Logged out -> "Log in to register" (redirects to login).
    - State 2: Logged in + Not registered + Free event -> "Register for free" button.
    - State 3: Logged in + Already registered -> "View ticket" link.
    - Connect the CTA strictly to the `POST /api/events/[id]/register` endpoint.
    - Show capacity indicators (e.g. "Only 5 spots left") if limited capacity.
  </action>
  <verify>CTA conditionally renders based on session and registration state.</verify>
  <done>User can click CTA and successfully register.</done>
</task>
