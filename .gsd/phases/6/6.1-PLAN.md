# Plan 6.1: Promo Codes Schema & Management API

## Objective
Create the schema for Promo Codes and provide an API for organizers to create and manage them for their events.

## Context
- .gsd/phases/6/SPEC.md
- database/event.model.ts

## Tasks

<task type="auto">
  <name>PromoCode Schema</name>
  <files>database/promo-code.model.ts</files>
  <action>
    Create a new Mongoose schema `PromoCode` with fields:
    - `eventId`: ObjectId ref to Event
    - `code`: String, uppercase, unique per event
    - `discountType`: Enum ['PERCENTAGE', 'FIXED']
    - `discountValue`: Number
    - `maxUses`: Number (optional)
    - `currentUses`: Number (default 0)
    - `validFrom`: Date
    - `validUntil`: Date
    - `isActive`: Boolean (default true)
    Ensure compound index on `{ eventId: 1, code: 1 }` with unique constraint.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>PromoCode model is defined with correct types and indexes.</done>
</task>

<task type="auto">
  <name>PromoCode Management API</name>
  <files>
    app/api/organizer/events/[id]/promo-codes/route.ts
    app/api/organizer/events/[id]/promo-codes/[codeId]/route.ts
  </files>
  <action>
    - Implement GET and POST for `app/api/organizer/events/[id]/promo-codes/route.ts` to list and create promo codes.
    - Implement PATCH and DELETE for `app/api/organizer/events/[id]/promo-codes/[codeId]/route.ts` to toggle status or delete.
    - Require organizer authentication and verify organizer owns the event.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>API routes for promo code CRUD are implemented securely.</done>
</task>
