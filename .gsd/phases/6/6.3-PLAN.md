# Plan 6.3: Referral Links & Tracking Schema

## Objective
Update the models to track referrals and create the ability to generate unique referral links.

## Context
- .gsd/phases/6/SPEC.md
- database/registration.model.ts

## Tasks

<task type="auto">
  <name>Referral Tracking Schema</name>
  <files>database/referral.model.ts</files>
  <action>
    Create a `ReferralTracker` schema:
    - `eventId`: ObjectId ref to Event
    - `organizerId`: ObjectId ref to User (creator of link)
    - `referralCode`: String, unique identifier for link
    - `name`: String (e.g., "Twitter Campaign")
    - `clicks`: Number (default 0)
    - `conversions`: Number (default 0)
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>ReferralTracker model created.</done>
</task>

<task type="auto">
  <name>Registration Referral Attribution</name>
  <files>
    database/registration.model.ts
    app/api/checkout/create-session/route.ts
    app/api/webhooks/stripe/route.ts
  </files>
  <action>
    - Add `referralId` (ObjectId) or `referralCode` (String) to `Registration` model to track source.
    - During `/api/checkout/create-session`, accept a `ref` parameter from the client.
    - Pass the `ref` in Stripe Session metadata.
    - In webhook, when creating registration, save the `referralCode` and increment `conversions` on the `ReferralTracker` model.
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>Registration correctly captures and records referrals.</done>
</task>

## Success Criteria
- [ ] Referral schema is defined.
- [ ] Checkout captures the referral code from the URL and persists it to the registration.
