# Plan 5.1: Infrastructure & Stripe Connect Onboarding

## Objective
Establish the financial database schema and implement the initial Stripe Connect Express onboarding flow for organizers.

## Context
- `database/organizer-profile.model.ts`
- `lib/validations/`
- `app/organizer/settings/payouts/page.tsx`

## Tasks

<task type="auto">
  <name>Implement Financial Data Models</name>
  <files>
    - database/ticket-type.model.ts
    - database/order.model.ts
    - database/payment-transaction.model.ts
  </files>
  <action>
    - **TicketType**: `eventId` (indexed), `name`, `price` (cents), `currency`, `quantityTotal`, `quantitySold` (atomic), `salesStartAt`, `salesEndAt`, `status` (active, sold_out, archived).
    - **Order**: `eventId` (indexed), `buyerUserId` (indexed), `status` (pending_payment, paid, failed, refunded), `lineItems` (snapshotted), `pricingSnapshot` (subtotal, platformFee, total), `stripeCheckoutSessionId` (unique index), `stripePaymentIntentId` (index).
    - **PaymentTransaction**: `orderId`, `type` (payment_captured, refund_issued, chargeback), `amount`, `externalRef` (Stripe ID), `occurredAt`.
  </action>
  <verify>Mongoose models exist and are exported correctly.</verify>
  <done>Database infrastructure ready for paid transactions.</done>
</task>
